# App Icons: Application Branding

![rs-appicons.png](/assets/project-theme/rs-appicons.png)

Visual branding plays an important role in application identity. The **App Icons** screen allows you to define the primary graphical elements that represent your application across the interface and in the browser.

This configuration ensures your generated application reflects your organization’s branding without manual file replacement.

---

## App Logo

The **App Logo** is the main branding image used within the application interface.

It typically appears in the navigation bar or header area and helps users visually identify the system. Clicking **Choose File** allows you to upload an image file from your system. Once selected, RadSystems automatically uses this image in the application layout.

For best results, use a clear, high-resolution image with a transparent background where possible.

---

## Favicon

The **Favicon** is the small icon displayed in browser tabs, bookmarks, and shortcuts.

Uploading a favicon helps users quickly recognize your application when multiple browser tabs are open. Use **Choose File** to select the favicon image. This is usually a small square icon designed for browser display.

---

## Saving Changes

Click **Okay** to save the selected icons. The new branding will be applied the next time the project is published.
